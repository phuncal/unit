# Unit 自动更新使用指南

## 功能概述

Unit 已完整集成自动更新功能，基于 GitHub Releases 实现。用户无需手动下载新版本，应用会自动检测并提示更新。

---

## 用户使用流程

### 自动检查更新

应用启动 5 秒后，会在后台静默检查 GitHub Releases 是否有新版本。

### 发现新版本

当检测到新版本时，应用顶部会显示**绿色提示条**：

```
┌─────────────────────────────────────────────────┐
│ 🔄 发现新版本 v1.0.1    [立即更新]    [×]      │
└─────────────────────────────────────────────────┘
```

用户可以：
- **点击"立即更新"**: 开始下载更新
- **点击 [×]**: 暂时关闭提示（下次启动仍会检查）

### 下载进度

点击"立即更新"后，提示条会显示下载进度：

```
┌─────────────────────────────────────────────────┐
│ 🔄 正在下载更新... 45%                          │
│ ████████████░░░░░░░░░░░░░░░                    │
└─────────────────────────────────────────────────┘
```

### 安装更新

下载完成后，提示条会变为：

```
┌─────────────────────────────────────────────────┐
│ ✓ 更新已准备就绪！  [重启并安装]                │
└─────────────────────────────────────────────────┘
```

点击"重启并安装"后，应用会自动：
1. 退出当前应用
2. 安装新版本
3. 重新启动

---

## 手动检查更新

如果不想等待自动检查，也可以手动触发：

1. 打开 Unit
2. 点击右上角 **设置** 图标
3. 滚动到底部的"关于"部分
4. 点击 **检查更新** 按钮

---

## 技术细节

### 更新源配置

- **Provider**: GitHub Releases
- **仓库地址**: https://github.com/phuncal/unit
- **检查频率**: 每次启动应用时检查一次

### 构建配置

- **架构**: Universal（同时支持 Intel 和 Apple Silicon Mac）
- **格式**: 
  - `.dmg` (安装镜像)
  - `.zip` (压缩包，供自动更新使用)

### 安全性

- ✅ 只从官方 GitHub Releases 下载更新
- ✅ 自动验证文件完整性（使用 blockmap 增量更新）
- ✅ 用户必须手动点击才会下载/安装，无静默安装

## 开发者发布新版本

详见 [RELEASE.md](./RELEASE.md) 文件。

简要流程：
1. 修改 `package.json` 的版本号
2. 运行 `npm run build:mac` 构建
3. 在 GitHub 创建 Release，上传生成的文件
4. 用户下次启动应用自动检测到更新

## 常见问题

### Q: 如何禁用自动更新检查？

目前不支持完全禁用，但可以：
- 每次出现提示时点击 [×] 关闭
- 不点击"立即更新"则不会下载

### Q: 更新会保留我的数据吗？

是的！更新只替换应用程序，不会影响：
- 对话记录（存储在 IndexedDB）
- 设置信息（存储在本地）
- 绑定的项目目录

### Q: 更新失败怎么办？

如果自动更新失败，可以：
1. 去 https://github.com/phuncal/unit/releases
2. 下载最新的 `.dmg` 文件
3. 手动重新安装

### Q: 提示条显示后找不到了？

提示条在应用顶部。如果点击了 [×] 关闭，下次启动应用会重新显示。

### Q: 为什么我看不到更新提示？

可能原因：
- 你已经是最新版本
- 网络连接问题，无法访问 GitHub
- 开发环境运行（只有打包后的应用会检查更新）
