核心功能

专为长期讨论而设计。

每一个功能的存在，都是因为深度创作工作需要它， 而不是因为可以加进来。

01

活档案

讨论结论持久保存在本地 archive.md 文件中。同一个项目目录下新建对话，思考会从上次结束的地方接续，不会遗失。

02

锚点消息

将关键内容标记为锚点——参考文档、核心前提——无论对话多长，它始终随每次请求携带，不被滑动窗口挤掉。

03

Token 感知

发送前查看费用预估。追踪每个对话的累计花费。滑动上下文窗口让长时间讨论保持经济可控。

04

任意 API，任意模型

接入 Anthropic、OpenRouter 或任何 OpenAI 兼容接口。自动获取可用模型列表。API Key 仅存本地，由系统钥匙串加密。

05

导出 Markdown

长内容直接保存为 .md 文件。文学策划转换为结构化技术文档。版本内容干净合并为新文件。

06

对话模板

将 System Prompt 保存为可复用的角色——游戏策划顾问、世界观构建者、叙事分析师。立刻进入深度讨论状态。

设计理念

「好的设计，
是尽可能少
的设计。」

— Dieter Rams

01

没有多余的东西

没有云同步、没有账号、没有数据追踪、没有联网搜索、没有插件系统。Unit 只做一件事，并保护完成这件事所需要的专注力。

02

严格隔离

每个对话独立存在，项目之间不互相污染。AI 只能看到你主动选择展示给它的内容。

03

你的数据，你的机器

所有内容存在你自己的文件夹里。API Key 由系统钥匙串加密保存。离开你电脑的，只有你主动选择发送的那条消息。

04

为持久工作而建

虚拟滚动处理数千条消息。流式输出从不阻塞。大文件导出从不卡死。长时间的工作值得一个可靠的工具。
